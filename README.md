## 🌟 Overview

CropCopilot answers two types of agriculture questions automatically:

| Query Type | Example | Handled By |
|---|---|---|
| **Conceptual / Knowledge** | "Which crops suit sandy soil?" | RAG Tool (ChromaDB + NVIDIA Embeddings) |
| **Data / Analytical** | "Compare average rainfall across all crops" | Text-to-SQL Tool (LLM → SQLite) |

The **CrewAI agent** intelligently decides which tool (or combination of tools) to invoke, synthesizes results, and returns a well-formatted, data-backed response.

## ✨ Features

- 🤖 **Agentic Orchestration** — CrewAI agent dynamically selects the right tool per query
- 📚 **RAG Knowledge Retrieval** — ChromaDB vector store with NVIDIA `nv-embedqa-e5-v5` embeddings
- 🗄️ **Text-to-SQL** — Converts natural language to SQLite queries via `meta/llama-3.1-70b-instruct`
- 🌐 **Modern Web UI** — Glassmorphism-styled frontend served via FastAPI
- 📊 **RAGAS Evaluation** — Measures Faithfulness, Answer Relevancy, and Context Precision
- 🔒 **Secure Config** — API keys managed via `.env` (never committed to source control)

## ⚙️ Setup & Installation

### Prerequisites

- Python **3.10+**
- An **NVIDIA NIM API key** (get one free at [build.nvidia.com](https://build.nvidia.com))

### 1. Clone the Repository

```bash
git clone https://github.com/Yuvarajsel/CropCopilot.git
cd CropCopilot
```

### 2. Create a Virtual Environment

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables

Create a `.env` file in the project root:

```env
NVIDIA_API_KEY=your_nvidia_nim_api_key_here
```

### 5. Ingest Data (One-Time Setup)

This step downloads the `Mahesh2841/Agriculture` dataset from HuggingFace, builds the SQLite database, and prepares the RAG document corpus:

```bash
python data_ingestion.py
```

> ✅ Expected output includes confirmation of SQLite table creation and JSON document generation.

## 🚀 Running the Application

```bash
python main.py
```

Then open your browser at **[http://127.0.0.1:8000](http://127.0.0.1:8000)**.

The vector store initializes automatically on first query (ChromaDB indexes the documents).

## 📊 Evaluation Pipeline

The system includes a comprehensive **RAGAS evaluation** to validate RAG quality:

```bash
python evaluate.py
```

**Metrics evaluated:**

| Metric | Description |
|---|---|
| **Faithfulness** | Are answers grounded in the retrieved context? |
| **Answer Relevancy** | How relevant is the answer to the question? |
| **Context Precision** | How precise is the retrieved context to the query? |

Results are saved to `data/ragas_evaluation_results.csv`.

## 🌱 Dataset

- **Crops covered:** rice, maize, chickpea, kidneybeans, pigeonpeas, mothbeans, mungbean, blackgram, lentil, pomegranate, banana, mango, grapes, watermelon, muskmelon, apple, orange, papaya, coconut, cotton, jute, coffee

## 🔍 How It Works

### Step 1 — Data Ingestion
`data_ingestion.py` downloads the HuggingFace dataset and:
- Stores tabular data in **SQLite** for structured queries
- Converts each crop record into a natural-language sentence, saved as JSON for **RAG indexing**

### Step 2 — Vector Store Setup
On first query, `rag_tool.py` loads the JSON documents, generates embeddings via NVIDIA's `nv-embedqa-e5-v5` model, and persists them in **ChromaDB**.

### Step 3 — Agent Reasoning
The **CrewAI agent** (`agri_agent.py`) receives your query, decides between:
- `retrieve_agri_info` → semantic search in ChromaDB
- `query_agri_database` → LLM generates SQL → executed against SQLite

### Step 4 — Response Synthesis
The agent combines tool outputs and generates a final, human-readable, data-backed response via `meta/llama-3.1-70b-instruct`.
